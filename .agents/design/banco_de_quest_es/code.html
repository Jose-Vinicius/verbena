Write-After-Write (WAW) and Write-After-Read (WAR) pseudo-hazards, enabling out-of-order execution with in-order commit.

Control hazards resulting from deeply nested branch instructions.

lightbulb Rationale

The Reorder Buffer (ROB) allows instructions to commit their results in programmatic order, despite executing out of order. By holding results temporarily before committing to the architectural register file, it resolves WAW and WAR dependencies (name dependencies) through register renaming, ensuring precise exceptions and correct state management.

lightbulb Rationale

The von Neumann architecture uses a unified memory space and bus system for both instructions and data, leading to the "von Neumann bottleneck." Harvard architecture separates these, allowing simultaneous fetching of instructions and data.

lightbulb Rationale

A Dirty Read happens specifically under the READ UNCOMMITTED isolation level. If Transaction A updates a row, and Transaction B reads that row before A commits, B has performed a dirty read. If A subsequently rolls back, B is left with data that "never existed" in the committed database state.
